# Phase 2.1 — Config Generation Gaps

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close gaps between current cheburbox config generation and real-world sing-box configs by adding per-user VLESS flow, inbound listen address, TLS ALPN, outbound domain_resolver, route default_domain_resolver, and DNS rules passthrough.

**Architecture:** Extend `config.Inbound` to support per-user flow via a new `InboundUser` type replacing the `[]string` users field. Add `Listen`, `ALPN` fields to `config.InboundTLS`. Add `DomainResolver` to `config.Outbound`. Add `DefaultDomainResolver` to `config.Route`. Wire DNS rules passthrough in `ConvertDNS`. Update persistence extraction for per-user flow. All changes are additive — no breaking API changes.

**Tech Stack:** `github.com/sagernet/sing-box/option` (VLESSUser.Flow, ListenOptions.Listen, InboundTLSOptions.ALPN, DomainResolveOptions, DNSOptions.Rules), `github.com/sagernet/sing/common/json/badoption` (Addr, Listable).

**Design Decisions:**
- Per-user flow is a config-time property, not a generated secret. It comes from `cheburbox.json`, not from persistence. Flow is not persisted — it is not a credential.
- The `Users` field on `config.Inbound` changes from `[]string` to `[]InboundUser` (breaking for cheburbox.json consumers, but this is pre-1.0). Each `InboundUser` has `Name` and optional `Flow`.
- `DomainResolveOptions` on outbound and route is modeled as a simple `*string` in cheburbox.json — just the DNS server tag. If non-empty, it becomes `&option.DomainResolveOptions{Server: tag}`. Full `DomainResolveOptions` fields (strategy, disable_cache, etc.) can be added later if needed.
- `Listen` is `string` in cheburbox.json (e.g., `"::"`, `"0.0.0.0"`), converted to `*badoption.Addr` via `netip.ParseAddr`.
- DNS rules continue as raw JSON passthrough (like route rules and route rule-sets) — they require sing-box registry context for unmarshaling.

```go
import (
	"net/netip"
	// ... existing imports

func parseListenAddr(s string) *badoption.Addr {
	if s == "" {
		return nil
	}
	addr, err := netip.ParseAddr(s)
	if err != nil {
		return nil
	}
	return (*badoption.Addr)(&addr)
}
```

Note: `badoption.Addr` is an alias for `netip.Addr`. The sing-box JSON marshaling treats `*badoption.Addr` as an optional address field.

```go
func buildVLESSInbound(in config.Inbound, creds InboundCredentials) (option.Inbound, error) {
	users := buildVLESSUsers(creds)
	if len(users) == 0 && len(in.Users) > 0 {
		return option.Inbound{}, errors.New("missing credentials for declared users")
	}

	opts := option.VLESSInboundOptions{
		ListenOptions: option.ListenOptions{
			Listen:     parseListenAddr(in.Listen),
			ListenPort: intToUint16(in.ListenPort),
		},
		Users: users,
	}

	if in.TLS != nil {
		opts.InboundTLSOptionsContainer.TLS = buildInboundTLS(in.TLS, creds)
	}

	return option.Inbound{
		Type:    in.Type,
		Tag:     in.Tag,
		Options: &opts,
	}, nil
}
```

```go
func ConvertRoute(route *config.Route) (*option.RouteOptions, error) {
	if route == nil {
		return &option.RouteOptions{}, nil
	}

	ctx := include.Context(context.Background())

	routeOpts := &option.RouteOptions{
		Final:               route.Final,
		AutoDetectInterface: route.AutoDetectInterface,
	}

	if route.DefaultDomainResolver != "" {
		routeOpts.DefaultDomainResolver = &option.DomainResolveOptions{
			Server: route.DefaultDomainResolver,
		}
	}

	if len(route.Rules) > 0 {
		var rules []option.Rule
		if err := singjson.UnmarshalContext(ctx, route.Rules, &rules); err != nil {
			return nil, fmt.Errorf("unmarshal rules: %w", err)
		}
		routeOpts.Rules = rules
	}

	if len(route.RuleSets) > 0 {
		var ruleSets []option.RuleSet
		if err := singjson.UnmarshalContext(ctx, route.RuleSets, &ruleSets); err != nil {
			return nil, fmt.Errorf("unmarshal rule_sets: %w", err)
		}
		routeOpts.RuleSet = ruleSets
	}

	for _, name := range route.CustomRuleSets {
		rs := option.RuleSet{
			Type: C.RuleSetTypeLocal,
			Tag:  name,
			LocalOptions: option.LocalRuleSet{
				Path: filepath.Join("rule-set", name+".srs"),
			},
		}
		routeOpts.RuleSet = append(routeOpts.RuleSet, rs)
	}

	return routeOpts, nil
}
```

In `generate/dns.go`, update `ConvertDNS` to unmarshal and include DNS rules. DNS rules use sing-box registry context (like route rules), so they must be unmarshaled with `include.Context`:

```go
import (
	"context"
	"fmt"

	C "github.com/sagernet/sing-box/constant"
	"github.com/sagernet/sing-box/include"
	"github.com/sagernet/sing-box/option"

	"github.com/Arsolitt/cheburbox/config"
)

func ConvertDNS(cfg config.DNS) (*option.DNSOptions, error) {
	servers, err := convertDNSServers(cfg.Servers)
	if err != nil {
		return nil, fmt.Errorf("convert dns servers: %w", err)
	}

	opts := &option.DNSOptions{
		RawDNSOptions: option.RawDNSOptions{
			Final:   derefOrEmpty(cfg.Final),
			Servers: servers,
			DNSClientOptions: option.DNSClientOptions{
				Strategy: parseDomainStrategy(derefOrEmpty(cfg.Strategy)),
			},
		},
	}

	if len(cfg.Rules) > 0 {
		ctx := include.Context(context.Background())
		var rules []option.DNSRule
		if err := singjson.UnmarshalContext(ctx, cfg.Rules, &rules); err != nil {
			return nil, fmt.Errorf("unmarshal dns rules: %w", err)
		}
		opts.Rules = rules
	}

	return opts, nil
}
```

Note: this requires adding `"context"` and `"github.com/sagernet/sing-box/include"` to the imports, and `singjson "github.com/sagernet/sing/common/json"` if not already present.
